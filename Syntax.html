Basic Syntax

Import in Swift - You can use the import statement to import any Objective C framework into the Swift program. To create a program for OS X playground we need to import Cocoa. While to create the same program for iOS playground then import UIKit.

Token in Swift - A Swift program consists of various tokens and a token is either a keyword, an identifier, a constant, a string literal, or a symbol.

Comments - Comments are like helping texts in your Swift program. They are ignored by the compiler. Multi-line comments start with /* and terminate with the characters */. Single-line comments are written using // at the beginning of the comment.

Semicolons - Swift does not require you to type semicolon(;) after each statement.Even if you use semicolon the compiler does not complain about it. However if using multiple statements in the same then semicolon can be used as delimiter.

Identifiers - A Swift identifier is a name used to identify a variable, function, or any other user-defined item. An identifier starts with an alphabet A to Z or a to z or an underscore _ followed by zero or more letters, underscores, and digits (0 to 9). Swift does not allow special characters such as @, $, and % within identifiers. Swift is a case sensitive programming language.

Keywords - Swift has some reserved keywords similar to C.These reserved keywords should not be used as constants or variables.

White spaces - A line containing only white spaces is known as a  blank line and the compiler totally ignores it.It is a term in Swift to define blanks, tab, newline characters and comments.

Literals - Literals are source code representation of a value of an integer, floating point number or a string type.
